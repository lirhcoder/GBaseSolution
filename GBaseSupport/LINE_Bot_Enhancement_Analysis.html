3. 会員証・ポイント統合の実装アプローチ

重要な前提

SC会員システムは各社独自開発が多く、直接API連携は現実的ではない（SC会員システム分析レポート参照）。 そのため、間接的なアプローチでユーザー体験を向上させる戦略を採用。

A

アプローチA: LINE公式アカウント活用

既存インフラ最大活用
  • SC公式LINE アカウントの友だち追加を入口に
  • Rich Menuで各機能への導線を提供
  • 会員証表示はLIFFアプリまたは外部リンクで対応
  • ポイント照会は「アプリでご確認ください」＋リンク
  • AI応答で会員制度の一般情報を即時提供
メリット: 開発コスト最小、即時導入可能
B

アプローチB: LINEミニアプリ連携

将来拡張オプション
  • LINE ミニアプリ（LIFF）でデジタル会員証を実装
  • バーコード/QRコード表示でレジ連携
  • ポイント残高表示（SC側システム連携が必要）
  • クーポン発行・利用機能
  • 会員ランクに応じたパーソナライズ
注意: SC会員システムとの個別連携開発が必要

5. Rich Menu + GBase Support 連携設計

# Rich Menu設定例（6分割）
RichMenu = {
    "size": { "width": 2500, "height": 1686 },
    "areas": [
        {
            # 左上: AIチャット起動
            "action": {
                "type": "message",
                "text": "AIに質問する"  # → GBase Support起動
            }
        },
        {
            # 中央上: フロアマップ
            "action": {
                "type": "uri",
                "uri": "https://floormap.digital/sc-name"  # Metamap連携
            }
        },
        {
            # 左下: 会員証表示
            "action": {
                "type": "uri",
                "uri": "https://line.me/R/nv/camera/scan"  # or LIFF/会員アプリ
            }
        },
        {
            # 中央下: ポイント確認
            "action": {
                "type": "message",
                "text": "ポイントについて教えて"  # → AI応答
            }
        }
    ]
}

# AI応答テンプレート（会員・ポイント関連）
def handle_member_query(query):
    if "ポイント" in query:
        return """
        ポイント残高の確認方法：
        1. SC公式アプリを開く
        2. マイページ → ポイント履歴

        ※ ポイント有効期限にご注意ください
        [アプリを開く] ← 外部リンクボタン
        """

    if "会員証" in query:
        return """
        デジタル会員証の表示：
        [会員証を表示する] ← LIFF or 外部アプリ

        お買い物の際にレジでご提示ください。
        """

結論と推奨事項

LINE Bot強化は開発コスト最小・効果最大の施策。 専用アプリ開発（数百万〜数千万円）なしで、日本最大のコミュニケーションプラットフォームを活用できる。

推奨アプローチ

  1. Rich Menu最適化（即座）
  2. 会員FAQ強化（即座）
  3. プッシュ通知導入（Q2）
  4. LIFF会員証（顧客要望時）

避けるべきアプローチ

  • 初期から会員システム直接連携を約束
  • 専用アプリ開発の提案
  • ポイント残高API連携の早期実装

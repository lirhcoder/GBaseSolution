# SC協会299社 系統調査計画

## 1. 現状の問題点

現在のCompany_Analysis_Matrixは**シミュレーションデータ**に基づいており、以下の問題がある：

| 問題点 | 説明 |
|--------|------|
| 企業名が不完全 | 299社中、実名が入っているのは約100社のみ |
| 地域データが虚偽 | ランダム生成された地域情報 |
| 機能ニーズが推測 | 業種から推測しただけで、実際の調査なし |
| スコアリングが恣意的 | 明確な根拠なく算出 |

## 2. 調査目的

1. **299社の完全な企業リスト**を確立する
2. 各企業の**実際のデジタル化状況**を把握する
3. **客観的な根拠**に基づいた機能ニーズを判定する
4. **獲得優先度スコア**を再計算する

## 4. 調査実施計画

### Phase 1: 会員名簿完全化（1-2日）
**目標**: 299社の正式名称と基本情報を確定

1. SC協会公式サイトから会員名簿PDFを取得
2. 299社の正式社名を全てリスト化
3. 15カテゴリへの分類を確認

**出力**: `299社基本リスト.csv`

### Phase 2: 公式サイト調査（3-5日）
**目標**: 各社のデジタル化状況を調査

調査対象: 299社全て（ただし効率化のため）
- Tier 1候補（約50社）: 詳細調査
- Tier 2候補（約100社）: 標準調査
- Tier 3候補（約150社）: 簡易調査

**調査方法**:
```
1. 公式サイトにアクセス
2. チェックリストに基づき確認（5-10分/社）
3. スクリーンショットを証拠として保存
4. 調査結果をスプレッドシートに記録
```

**出力**: `299社デジタル化調査.csv`

### Phase 3: スコアリング再計算（1日）
**目標**: 客観的根拠に基づくスコア算出

新スコアリングモデル:
```
総合スコア =
  企業規模スコア (25%) +
  機能ニーズスコア (30%) +
  インバウンド関連度 (20%) +
  導入ハードル (15%) +
  戦略的価値 (10%)
```

各スコアの根拠:
- **企業規模**: 運営SC数、売上規模（公開情報）
- **機能ニーズ**: Phase 2の調査結果から計算
- **インバウンド**: 立地、免税店、多言語サイト
- **導入ハードル**: 既存システム有無、企業規模
- **戦略的価値**: 業界影響力、グループ展開可能性

**出力**: `299社スコアリング結果.csv`

### Phase 4: HTML更新（1日）
**目標**: Company_Analysis_Matrixを実データで更新

1. CSVデータをJavaScript配列に変換
2. 根拠リンク（公式サイト）を追加
3. 機能ニーズの判定根拠を表示
4. エクスポート機能の改善

## 7. 実施オプション

### Option A: 完全調査（推奨）
- 299社全てを調査
- 所要時間: 約7-10日
- 精度: 高

### Option B: サンプル調査 + 推定
- 各セグメントから代表企業を詳細調査（約50社）
- 残りはセグメント特性から推定
- 所要時間: 約3-5日
- 精度: 中

### Option C: 段階的調査
- まずTier 1候補50社を詳細調査
- 成果を見ながらTier 2, 3を順次調査
- 所要時間: 初回2-3日、以降随時
- 精度: 高（調査済み部分）

   - https://www.jcsc.or.jp/ から会員リストを取得

   - Option A/B/Cのいずれかを決定

3. **調査テンプレートの準備**
